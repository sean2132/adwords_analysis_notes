# Python 阶段学习总结

## ETL / 数据工厂｜可持续更新

> **项目：** Adword / SEO Data Analysis  
> **阶段定位：** 将原始 Excel/CSV 数据通过 Python 自动化清洗、标准化、建模拆表、输出可复用数据集（供 MySQL/Power BI 使用）  
> **核心关键词：** ETL、可复现、数据质量、事实表/维表、星型模型、质量门禁  

---

## 1. 本阶段应掌握的总体能力

- 将"Excel 里人工检查与修正"升级为"Python 可复跑的 ETL 流程"
- 读数据后先做**数据体检（Profiling）**，再做**清洗（Transform）**，最后**输出（Load）**
- 能将数据字典（字段含义/类型/范围/空值策略）落地为代码规则
- 能为后续数据库与 BI 建模准备数据：
  - 拆事实表/维表
  - 生成代理主键
  - 保证关系可用
- 具备最基本的数据工程思维：
  - 路径可移植
  - 步骤可复现
  - 结果可验证
  - 异常可追踪

## 2. 背后的数据分析思维（为什么要这样做）

### 2.1 可复现（Reproducible）

- 同样输入 → 同样输出
- 不依赖手工操作，规则固化在代码中

### 2.2 先体检再清洗（Profiling before Transform）

- 先明确：字段类型是否正确、缺失/重复/异常规模多大、哪些列有风险
- 避免"没搞清数据就开始建模/画图"的结论偏差

### 2.3 先标准化再关联（Normalize before Join）

- 文本/日期/数值先统一，才能稳定去重、合并、入库、建关系
- 否则 Join 失败、重复统计、BI 关系不生效会频繁发生

### 2.4 建模思维：事实表 + 维表（Star Schema）

- **事实表**：承载"度量与事件"（Traffic、Cost…）
- **维表**：承载"实体与属性"（Keyword…）并提供唯一主键
- 为 MySQL 索引与 Power BI 模型关系打基础

### 2.5 质量门禁（Fail Fast）

- 通过断言/校验让错误尽早暴露
- 避免 **silent error**（悄悄错了但不报错，后面图做出来也不可信）

## 3. 类似项目的 Python 操作步骤（通用 SOP）

### Step 1：Extract（抽取/读取）

- 明确数据源：Excel / CSV / DB / API
- **路径管理**：统一项目目录结构、避免写死绝对路径
- **读取前探查**：确认 sheet / 列名 / 编码等
- **读取后验证**：行列数、样本数据、列类型

### Step 2：Profiling（数据体检/探索）

| 检查项 | 内容 |
|--------|------|
| 查看结构 | 列名、dtype、非空数、内存 |
| 查看缺失 | 每列缺失数与缺失率 |
| 查看重复 | 整行重复、业务主键重复 |
| 查看范围 | 比例/排名/难度/竞争度等是否越界 |
| 查看分布 | 极值与数量级是否合理（sanity check） |

### Step 3：Transform（清洗/标准化）

| 类型 | 操作 |
|------|------|
| 文本 | 去空格、去不可见字符、大小写统一、空字符串归一 |
| 数值 | 强制转数值，失败转缺失（NaN） |
| 日期 | 强制转日期，失败转缺失（NaT） |
| 量纲 | 百分比字段统一为 0–100 或 0–1（必须先判断再统一） |
| 异常 | 按规则截断（clip）/ 置空（NaN）/ 标记异常 |
| 去重 | 按业务主键定义去重（Keyword / Title+Keyword 等） |

### Step 4：Model（拆表建模）

- **维表**：提取唯一实体（去重）+ 生成代理主键（整数 key）
- **事实表**：保留度量字段 + 外键（代理主键）
- **回写外键**：把维表 key 合并回事实表

### Step 5：Load（输出/入库）+ 校验

- 输出 CSV（或直接写 MySQL）
- 校验外键完整性、维表主键唯一性、输出行数变化可解释
- 固化输出结构（字段白名单），避免临时列污染下游

## 4. 关键库与常用功能

### 4.1 Jupyter / Notebook（.ipynb）

- **作用**：交互式运行、按单元格验证、保留输出、可写说明
- **使用要点**：每个 ETL 步骤都应有"读取 / 处理 / 验证"闭环

---

### 4.2 pathlib（路径管理）

| 方法 | 说明 |
|------|------|
| `Path.cwd()` | 获取当前工作目录（Notebook 运行目录） |
| `.parent` | 上一级目录（用于定位项目根目录） |
| `/` | 路径拼接（跨平台，避免 Windows 转义问题） |
| `.exists()` | 检查文件/目录是否存在（防止 FileNotFound） |
| `.mkdir(parents=True, exist_ok=True)` | 创建目录（工程化输出必备） |

---

### 4.3 pandas（核心数据处理）

#### 读取 / 写出

| 方法 | 说明 |
|------|------|
| `read_excel` | 读取 Excel；关键参数：`sheet_name` |
| `ExcelFile + .sheet_names` | 读取前探查 sheet 名称 |
| `to_csv` | 导出 CSV；关键参数：`index`、`encoding` |

#### 数据体检（Profiling）

| 方法 | 说明 |
|------|------|
| `info` | 列类型 / 非空计数 / 内存 |
| `describe` | 统计摘要 |
| `isna / sum` | 缺失统计 |
| `duplicated` | 整行重复判断 |

#### 类型转换（Transform 基础）

| 方法 | 说明 |
|------|------|
| `to_numeric(errors="coerce")` | 转数值；失败变 NaN |
| `to_datetime(errors="coerce")` | 转日期；失败变 NaT |
| `astype` | 强制类型转换 |

#### 文本清洗

| 方法 | 说明 |
|------|------|
| `.str.strip()` | 去首尾空格 |
| `.str.lower()` | 统一小写 |
| `.str.replace(..., regex=True)` | 正则替换 |
| `.eq("")` | 判断空字符串 |

#### 规则清洗

| 方法 | 说明 |
|------|------|
| `clip(min, max)` | 范围截断 |
| `loc[条件, 列]` | 条件赋值 |
| `drop_duplicates(subset=...)` | 按业务主键去重 |
| `dropna / fillna` | 缺失处理 |

#### 合并与建模

| 方法 | 说明 |
|------|------|
| `merge(on=..., how=...)` | 表连接 |
| `sort_values` | 排序 |
| `reset_index(drop=True)` | 重置索引 |

---

### 4.4 numpy（数值与缺失）

- `np.nan`：数值缺失表示
- 缺失判断用 `isna()`，不要用 `== np.nan`

## 5. 重点知识点沉淀（经验清单）

| 知识点 | 核心结论 |
|--------|---------|
| `.ipynb` 是什么 | 交互式 Notebook 文件 |
| 各库职责边界 | pandas处理表格、numpy处理数值 |

| 为什么用代理主键 | 稳定关联、性能好 |
| 为什么做质量门禁 | 防止 silent error |

## 6. 常见坑与注意事项

| 坑 | 说明 |
|----|------|
| 路径硬编码 | 迁移失败 |
| 类型未显式转换 | 可能统计错误 |
| 百分比量纲不统一 | 结果差100倍 |
| 文本未标准化 | 关联失败 |
| 维表主键不唯一 | Power BI 多对多 |
| 输出编码问题 | Excel 乱码 |
