# 二、Power BI 的核心思维

Power BI 不是画图工具，而是：

> 数据模型 + 指标逻辑 + 可视化表达

---

## 1. 建模优先于可视化

顺序必须是：

1. 数据关系正确  
2. 度量值正确  
3. 再做图表  

如果模型错误：

- 筛选器失效
- 指标重复计算
- 图表数据异常

# 三、数据模型设计

## 1. 星型模型（Star Schema）

你在项目中构建了：

- ✅ 事实表（website_traffic_data）
- ✅ 维表（keyword）

关系：

keyword (1) —— (*) website_traffic_data

---

## 2. 关系设置要点

- 一对多关系（1:*）
- 单向过滤（从维表到事实表）
- 主键必须唯一
- 外键不能缺失

---

## 3. 常见问题总结

### ❌ 关系非活跃

原因：

- 多路径过滤
- 多对多关系
- 维表主键不唯一

解决：

- 删除冗余关系
- 保证维表唯一
- 采用单向过滤

---

# 四、DAX 度量值（Measure）体系

## 1. 为什么用 Measure？

- 动态计算
- 自动考虑筛选上下文
- 避免重复逻辑
- 可复用

---

## 2. 基础聚合函数

| 函数 | 作用 |
|------|------|
| SUM | 求和 |
| AVERAGE | 平均 |
| COUNT | 计数 |
| MAX/MIN | 最大/最小 |
| DIVIDE | 安全除法 |

---

## 3. DIVIDE 的意义

- 防止除以 0 报错
- 可指定默认值
- 推荐代替 `/`

---

## 4. 筛选上下文（Filter Context）

### ✅ 定义

当前页面/切片器/图表对数据施加的过滤条件。

### ✅ 理解

- Measure 会根据筛选动态变化
- 计算列不会动态变化

---

## 5. 行上下文 vs 筛选上下文

| 类型 | 用途 |
|------|------|
| 行上下文 | 计算列 |
| 筛选上下文 | 度量值 |

---

## 6. 常见 DAX 概念

- 聚合优先（先 SUM 再除）
- 不要逐行计算后再平均
- 比率类指标必须基于聚合结果

---

# 五、可视化设计逻辑

## 1. 仪表板结构设计

推荐结构：

1. KPI 区域（顶部）
2. 趋势分析（中部）
3. 结构分析（底部）
4. 筛选器区域（侧边或顶部）

---

## 2. 机会象限图设计

### 维度含义：

| 维度 | 含义 |
|------|------|
| X轴 | Keyword Difficulty |
| Y轴 | Search Volume |
| 气泡大小 | Total Traffic |

### 业务意义：

识别“高搜索量 + 低难度”的关键词机会。

---

## 3. 散点图注意事项

- 必须添加“详细信息/图例”字段，否则只有一个点
- 使用平均值而不是总和
- 添加常数线形成四象限

---

# 六、视觉优化原则

## 1. 数字单位统一

- 使用 K/M 简写
- 保留合理小数位
- 避免“17 千”混乱表达

---

## 2. 配色原则

- 主色不超过 2~3 种
- 强调色用于关键指标
- 避免过度彩色

---

## 3. 排版原则

- 左对齐
- 去除不必要网格线
- 保持视觉层级清晰

---

# 七、切片器与交互逻辑

- 统一控制所有图表

## 2. 类型选择

- 下拉式（节省空间）
- 范围型（数值/日期）

# 八、你在项目中遇到的问题总结

## 1. 只有一个散点

原因：

- 没有添加分组字段（Details/Legend）

解决：

- 添加 Keyword 作为分组

---

## 2. 百分比总和错误

原因：

- 使用 SUM 聚合属性字段

解决：

- 使用 AVERAGE

---

## 3. 单位显示不专业

原因：

- 默认中文单位显示“千”

解决：

- 设置单位
- 或自定义 Measure

---

## 4. 度量值分散

原因：

- 没有专门的 Measure 表

解决：

- 建立独立 Measure 表统一管理
