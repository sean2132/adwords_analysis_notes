# 三、建库与字符编码知识点

## 1. 创建数据库

### 核心知识点：

- `CREATE DATABASE`
- 设置 `CHARACTER SET`
- 设置 `COLLATE`

---

## 2. CHARACTER SET vs COLLATE

| 设置 | 作用 |
|------|------|
| CHARACTER SET utf8mb4 | 决定存储编码 |
| COLLATE utf8mb4_unicode_ci | 决定排序和比较规则 |

### ✅ 为什么用 utf8mb4_unicode_ci？

- 支持完整 Unicode
- 不区分大小写（ci = case insensitive）
- 适合关键词类数据

---

# 四、字段类型设计原则（重要）

数据库字段类型设计是 MySQL 阶段的核心能力。

## 1. 数值类型选择

| 类型 | 使用场景 |
|------|----------|
| INT | 排名、数量 |
| BIGINT | 超大数量 |
| DECIMAL | 金额、百分比 |
| FLOAT | 不推荐用于金额 |
| DATE | 日期 |
| VARCHAR | 文本 |

---

## 2. DECIMAL(10,2) 的含义

- 总共 10 位
- 其中 2 位小数
- 精确计算（适合金额）

### 为什么不用 FLOAT？

- 浮点数存在精度误差
- 金融数据必须精确

---

# 五、主键与外键设计

## 1. 主键（PRIMARY KEY）

- 唯一
- 自动索引
- 不允许为空

## 2. 外键（FOREIGN KEY）

- 保证数据一致性
- 防止事实表引用不存在的维表数据

### 设计原则：

> 维表必须唯一  
> 事实表必须有外键  

---

# 六、索引（Index）

## 1. 什么是索引？

- 类似书的目录
- 加速查询
- 底层通常为 B+Tree 结构

## 2. 什么时候建索引？

✅ JOIN 字段  
✅ WHERE 条件字段  
✅ GROUP BY 字段  
✅ 外键字段  

## 3. 索引的代价

- 占用空间
- 插入变慢
- 不要乱建

# 九、你在本项目中遇到的典型问题总结

## 1. 数据导入类型错误

### 问题：

字符串导入到 INT 列

### 原因：

- CSV 字段顺序与数据库字段顺序不匹配

### 解决：

- 检查字段映射
- 调整数据库结构

---

## 2. 重复列名错误（视图阶段）

### 问题：

Duplicate column name 'keyword'

### 原因：

- SELECT w.* 已包含 keyword
- 又单独 SELECT k.keyword

### 解决：

- 使用别名
- 或删除重复字段

---

## 3. TEXT 字段无法建索引

### 原因：

- 导入时类型错误

### 解决：

- 修改字段类型为 INT 或 VARCHAR
