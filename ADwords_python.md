\## 2. 背后的数据分析思维（为什么要这样做）



\### 2.1 可复现（Reproducible）



\- 同样输入 → 同样输出

\- 不依赖手工操作，规则固化在代码中



\### 2.2 先体检再清洗（Profiling before Transform）



\- 先明确：字段类型是否正确、缺失/重复/异常规模多大、哪些列有风险

\- 避免"没搞清数据就开始建模/画图"的结论偏差



\### 2.3 先标准化再关联（Normalize before Join）



\- 文本/日期/数值先统一，才能稳定去重、合并、入库、建关系

\- 否则 Join 失败、重复统计、BI 关系不生效会频繁发生



\### 2.4 建模思维：事实表 + 维表（Star Schema）



\- \*\*事实表\*\*：承载"度量与事件"（Traffic、Cost…）

\- \*\*维表\*\*：承载"实体与属性"（Keyword…）并提供唯一主键

\- 为 MySQL 索引与 Power BI 模型关系打基础



\### 2.5 质量门禁（Fail Fast）



\- 通过断言/校验让错误尽早暴露

\- 避免 \*\*silent error\*\*（悄悄错了但不报错，后面图做出来也不可信）

\## 3. 类似项目的 Python 操作步骤（通用 SOP）



\### Step 1：Extract（抽取/读取）



\- 明确数据源：Excel / CSV / DB / API

\- \*\*路径管理\*\*：统一项目目录结构、避免写死绝对路径

\- \*\*读取前探查\*\*：确认 sheet / 列名 / 编码等

\- \*\*读取后验证\*\*：行列数、样本数据、列类型



\### Step 2：Profiling（数据体检/探索）



| 检查项 | 内容 |

|--------|------|

| 查看结构 | 列名、dtype、非空数、内存 |

| 查看缺失 | 每列缺失数与缺失率 |

| 查看重复 | 整行重复、业务主键重复 |

| 查看范围 | 比例/排名/难度/竞争度等是否越界 |

| 查看分布 | 极值与数量级是否合理（sanity check）|



\### Step 3：Transform（清洗/标准化）



| 类型 | 操作 |

|------|------|

| 文本 | 去空格、去不可见字符、大小写统一、空字符串归一 |

| 数值 | 强制转数值，失败转缺失（NaN） |

| 日期 | 强制转日期，失败转缺失（NaT） |

| 量纲 | 百分比字段统一为 0–100 或 0–1（必须先判断再统一） |

| 异常 | 按规则截断（clip）/ 置空（NaN）/ 标记异常 |

| 去重 | 按业务主键定义去重（Keyword / Title+Keyword 等） |



\### Step 4：Model（拆表建模）



\- \*\*维表\*\*：提取唯一实体（去重）+ 生成代理主键（整数 key）

\- \*\*事实表\*\*：保留度量字段 + 外键（代理主键）

\- \*\*回写外键\*\*：把维表 key 合并回事实表



\### Step 5：Load（输出/入库）+ 校验



\- 输出 CSV（或直接写 MySQL）

\- 校验外键完整性、维表主键唯一性、输出行数变化可解释

\- 固化输出结构（字段白名单），避免临时列污染下游

\## 4. 关键库与常用功能



\### 4.1 Jupyter / Notebook（.ipynb）



\- \*\*作用\*\*：交互式运行、按单元格验证、保留输出、可写说明

\- \*\*使用要点\*\*：每个 ETL 步骤都应有"读取 / 处理 / 验证"闭环



\---



\### 4.2 pathlib（路径管理）



| 方法 | 说明 |

|------|------|

| `Path.cwd()` | 获取当前工作目录（Notebook 运行目录） |

| `.parent` | 上一级目录（用于定位项目根目录） |

| `/` | 路径拼接（跨平台，避免 Windows 转义问题） |

| `.exists()` | 检查文件/目录是否存在（防止 FileNotFound） |

| `.mkdir(parents=True, exist\_ok=True)` | 创建目录（工程化输出必备） |



\---



\### 4.3 pandas（核心数据处理）



\#### 读取 / 写出



| 方法 | 说明 |

|------|------|

| `read\_excel` | 读取 Excel；关键参数：`sheet\_name` |

| `ExcelFile + .sheet\_names` | 读取前探查 sheet 名称 |

| `to\_csv` | 导出 CSV；关键参数：`index`（是否导出索引）、`encoding`（避免乱码）|



\#### 数据体检（Profiling）



| 方法 | 说明 |

|------|------|

| `info` | 列类型 / 非空计数 / 内存；快速发现 dtype 不对 |

| `describe` | 统计摘要；`include="all"` 可包含文本列信息 |

| `isna / sum` | 缺失统计（缺失矩阵 → 统计数量） |

| `duplicated` | 整行重复判断；结合 `sum` 得到重复行数 |



\#### 类型转换（Transform 基础）



| 方法 | 说明 |

|------|------|

| `to\_numeric(errors="coerce")` | 转数值；失败变缺失（NaN）而不中断 |

| `to\_datetime(errors="coerce")` | 转日期；失败变缺失（NaT） |

| `astype` | 强制类型转换（常用于将列统一成字符串便于清洗） |



\#### 文本清洗（Series.str 系列）



| 方法 | 说明 |

|------|------|

| `.str.strip()` | 去首尾空格 |

| `.str.lower()` | 统一小写（避免大小写导致去重/关联失败） |

| `.str.replace(..., regex=True)` | 用正则替换（如合并多个空白字符） |

| `.eq("")` | 判断空字符串（区分"真缺失 NaN"与"假缺失 空字符串"） |



\#### 规则清洗与异常处理



| 方法 | 说明 |

|------|------|

| `clip(min, max)` | 范围截断（如百分比 0–100、Competition 0–1） |

| `loc\[条件, 列]` | 按条件定位并赋值（如将无效排名置空） |

| `drop\_duplicates(subset=...)` | 按业务主键去重（不只是整行去重） |

| `dropna / fillna` | 缺失处理（删除/填充策略需基于数据字典） |



\#### 合并与建模



| 方法 | 说明 |

|------|------|

| `merge(on=..., how=...)` | 表连接（等价 SQL JOIN）；常用于回写维表主键到事实表 |

| `sort\_values` | 排序（稳定输出、便于检查） |

| `reset\_index(drop=True)` | 重置索引（维表生成后常用） |



\#### 输出结构控制（工程习惯）



\- \*\*列白名单选择\*\*：只输出需要的字段，避免临时列进入下游

\- \*\*`.copy()`\*\*：避免视图引用导致的"改一处影响另一处"副作用



\---



\### 4.4 numpy（数值与缺失）



\- `np.nan`：数值缺失的标准表示

\- 与 pandas 缺失判断配合使用

\- ⚠️ 缺失不要用 `== np.nan` 判断，应使用 `isna()`

\## 5. 重点知识点沉淀（经验清单）



| 知识点 | 核心结论 |

|--------|---------|

| `.ipynb` 是什么 | Jupyter Notebook 文件，代码+说明+输出一体，适合分步验证 |

| 各库职责边界 | pandas处理表格、numpy处理数值/缺失、openpyxl读写xlsx、jupyter提供运行环境 |

| pathlib 的优势 | 用 `/` 拼路径，跨平台，避免 Windows `\\` 转义坑 |

| NaN/None/NULL 区别 | Python层/pandas层/SQL层各自的缺失表示，判断方式不同 |

| 为何用 `coerce` | 保证流程不中断，异常可统计，失败转缺失而非报错 |

| 百分比量纲判断 | 用数据分布比例判断是 0–1 还是 0–100，再统一 |

| 为何用代理主键 | 稳定关联、性能更好、建模更规范 |

| 为何做质量门禁 | 防止 silent error，确保下游数据可信 |

\## 6. 常见坑与注意事项



| 坑 | 说明 |

|----|------|

| 路径硬编码 | 导致迁移失败；推荐项目化目录 + pathlib |

| 类型未显式转换 | "看起来是日期/数字"不代表类型正确，必须显式转换并统计失败率 |

| 百分比量纲不统一 | 后续图表与结论差 100 倍 |

| 文本未标准化 | 去重失败、维表重复、Join 关联不上 |

| 维表主键不唯一 | Power BI 出现多对多关系、关系非活跃、筛选不传递 |

| 输出编码与索引 | 注意 `encoding` 与 `index=False`，否则 Excel/BI 乱码或多出无意义列 |
