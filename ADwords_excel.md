# Excel 阶段学习总结（可持续更新版）

> 项目：Adword / SEO 数据分析  
> 阶段目标：用 Excel **理解原始数据、建立数据字典、做数据质量诊断、制定清洗规则**  
> 注意：Excel 在本项目中更像“侦查与定义问题”的工具，而不是最终分析交付平台。

---

## 1. 本阶段你学到/应掌握的核心知识点

### 1.1 数据理解（Data Understanding）
- **字段识别与分类**
  - 维度字段（文本/类别）：`Title`, `Keyword`, `Trends` 等
  - 指标字段（数值）：`Search Volume`, `CPC`, `Traffic`, `Traffic Cost`, `Competition`, `Keyword Difficulty` 等
  - 时间字段（日期）：`Last Seen`
- **业务含义与指标口径**
  - `Position / Previous position`：排名类指标（不能求和）
  - `Traffic (%) / Traffic Cost (%)`：比例类指标（必须确认量纲是 `0~1` 还是 `0~100`）
  - `Competition`：通常应在 `0~1`
  - `Keyword Difficulty`：通常应在 `0~100`

### 1.2 数据字典（Data Dictionary / 数据说明书）
你建立了可复用的“字段说明表”，包含：
- 字段名（Field）
- 含义（Meaning）
- 期望类型（Expected Type）
- 合法范围（Valid Range）
- 是否允许为空（Nullability）
- 清洗策略（Cleaning Strategy）

**关键价值：**
- 让“清洗规则”有据可依，不靠感觉
- 后续 Python / SQL / BI 的建模与指标计算都能对齐口径
- 项目复盘、交接、复用的基础文档

### 1.3 数据质量检查（Data Quality Check）
你在 Excel 中学习了如何识别常见问题：
- **缺失值**：空白、空字符串、无法解析的日期/数字
- **重复值**：Keyword 重复、Title+Keyword 重复（视业务定义）
- **类型错误**：数字被当文本（如 `Search Volume` 带逗号）、日期是文本
- **范围异常**：
  - `Position <= 0`
  - `Competition > 1` 或 `<0`
  - `%` 字段越界或量纲不一致
  - `Keyword Difficulty > 100`
- **脏字符**：多余空格、不可见字符、大小写不统一

相关 Excel 技能点（你已用到或应继续熟练）：
- 结构化表（Table）与结构化引用（如 `[@Keyword]`）
- 公式清洗：`TRIM` / `CLEAN` / `LOWER`
- 日期识别与验证：`ISNUMBER` 判断真日期；必要时 `DATEVALUE`
- 去重逻辑：`COUNTIF` 判重（注意中文 Excel 常用 `;` 分隔参数）
- 拼接与合并：`TEXTJOIN`（用于把分散的内容合并回一个单元格）
- 透视表（Pivot Table）：快速理解分布、验证数量级是否合理

### 1.4 透视表用于“理解而非交付”
你做了两类典型透视表：
- `Keyword` 的计数：看关键词出现频率（识别头部词、重复情况）
- `Keyword Difficulty` 的平均值：看难度分布（识别难度高低、是否合理）

你理解了一个重要观念：
> 透视表不是最终报表，但能快速让你判断数据“像不像真的”，避免把明显错误的数据带到后续环节。

## 2. 在其他类似项目中（Excel 阶段）的通用操作步骤（模板）

> 下面是可复用 SOP：每次拿到“原始数据文件”，先按这个走一遍。

### Step 0：备份与记录
- 备份原始文件（只读保留）
- 建工作副本（可添加检查列、字典等）

### Step 1：快速扫一遍数据（5-10 分钟）
- 看表头、行数、列数
- 抽样看 10~20 行（有没有明显的错位/乱码/空列）

### Step 2：建立数据字典（你最重要的产出）
- 字段含义、类型、范围、可空性、清洗策略
- 明确哪些字段是“维度”、哪些是“指标”、哪些是“日期”

### Step 3：数据质量诊断（定位问题而不是修修补补）
至少做 4 类检查：
1) 缺失（Null / Blank / 无法解析）
2) 重复（按业务主键定义：Keyword？Title+Keyword？）
3) 类型（数字/日期是不是被当成文本）
4) 范围（排名、比例、难度、竞争度等）

### Step 4：分布与数量级 sanity check（透视表/描述统计）
- 计数、均值、最大最小、Top N
- 判断：是否出现“明显不合理”的数量级（例如 Traffic 极大但 CPC 为 0）

### Step 5：输出清洗规则草案（为 Python/SQL 准备）
- 哪些列需要 `trim/lower/clean`
- 日期怎么解析（格式不一致怎么处理）
- 缺失值策略（填 0、填均值、还是删除行）
- 异常值策略（截断、置空、标记异常）
- 是否要生成新字段（如 Keyword_Clean、分类字段、ID 等）

## 3. 背后的数据分析思维（Excel 阶段的“为什么”）

### 3.1 从“数据长什么样”到“数据能不能用”
Excel 阶段的核心不是算指标，而是回答：
- 这些字段是否可信？
- 指标口径是否一致？
- 哪些问题会导致后续分析结论失真？

### 3.2 “先定义，再计算”
你通过数据字典把问题结构化：
- 定义字段类型与范围 → 才能判断异常
- 定义缺失处理策略 → 才能一致地清洗
- 定义聚合方式（SUM/AVG/不汇总）→ 才能避免 BI 图表误导

### 3.3 让规则可复用、可自动化
Excel 阶段的检查列/透视表，其实是在“模拟未来 Python/SQL 会做的事”：
- 你不是为了在 Excel 把数据修好
- 而是为了明确后续自动化处理需要什么规则与边界条件

## 4. 本阶段的注意事项（踩坑清单）

- **仅改单元格格式 ≠ 数据类型真的变了**
  - 日期/数字如果本质是文本，改格式不会自动转换
  - 需要用验证（`ISNUMBER` / `+1`）确认
- **中文 Excel 参数分隔符常用 `;`**
  - 复制英文公式时经常需要把 `,` 改为 `;`
- **比例字段量纲必须统一**
  - 0~1 与 0~100 混用会导致所有图表和结论错 100 倍
- **排名类指标不要 SUM**
  - Position/Previous position 应 AVG 或不汇总
- **不要在 Excel 手工“修数据”太多**
  - 手工修会难以复现；真正清洗应交给 Python/SQL 流程
